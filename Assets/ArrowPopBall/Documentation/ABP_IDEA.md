### **1. 게임 기본 정보 확인**

1. **게임 이름 및 장르**:
    - **Arrow Pop Escape (가제)** / **하이퍼캐주얼 퍼즐 (ArrowEscape-like) + Target Pop(풍선 파괴)**
2. **게임 시점**:
    - **2D** (클린한 하이퍼캐주얼 스타일)
3. **카메라 방식**:
    - **탑다운(Top-down)** + **줌 인/아웃 & 스크롤(패닝)**
    - 단, **상단 목표(풍선)는 월드가 아니라 Screen Space UI로 고정**되어 항상 보이도록 설계
4. **플레이 방식**:
    - **수동 조작**(사용자 입력으로 화살표를 이동/탈출시키는 퍼즐)
5. **주요 조작 방식**:
    - **드래그/스와이프** 중심(화살표 이동/정렬/탈출 유도)
    - 보조: 탭(힌트, 리셋 등)
6. **주요 플랫폼 및 기기**:
    - **iOS / Android 모바일**(세로 화면 기준)
7. **사용 엔진**:
    - **Unity**

### **2. 게임 플레이 기본 구조**

**세션 플레이 흐름(초기 접속 → 종료)**:
    - 레벨 진입 → 퍼즐 영역(화살표) 관찰 → 화살표를 이동시켜 그리드 밖으로 “탈출(Extract)” → 상단 UI 풍선이 동일 색상 매칭으로 터짐 → 목표 달성 시 클리어/실패 시 재도전 → 다음 레벨
**인게임 루프(순서/주기)**:
    1. **화살표 경로/막힘 분석**
    2. **화살표 이동(그리드 내 충돌/제약 존재)**
    3. **그리드 밖으로 탈출(Extract) 트리거**
    4. **탈출 화살표가 가장 가까운 동일 색 풍선으로 호밍 → Pop**
    5. **남은 풍선/콤보 확인** → 다음 수 선택
    6. **클리어/실패 판정** → 반복
**평균 세션 길이(분)**:
- 목표: **3~7분/세션**(레벨 연속 진행 기준)
- 방향: “한 레벨 20~60초” 템포로 누적 플레이타임을 확보

### **3. 게임 시스템 정보 확인**

**게임의 핵심 루프**:
- **퍼즐(화살표 탈출) → 결과(상단 풍선 파괴) → 레벨 완료/재시도 → 진행**
**수집 콘텐츠**:
- MVP에서는 최소화 권장(선택)
- 확장 시: **화살표 스킨 / 풍선 스킨 / 이펙트(컨페티) 스킨** 정도의 라이트 컬렉션
**성장 콘텐츠**:
- MVP에선 **불필요/최소화** 권장
- 확장 시: “콤보 유지”, “특수 화살표 해금(관통/분열/폭발)” 같은 **경량 성장/언락**이 적합
 **광고·과금 모델**:
- 기본: **IAA 중심**
- **보상형 광고**: 힌트(추천 화살표 하이라이트), 실패 후 1회 구제
- **전면 광고(Interstitial)**: 레벨 완료 n회당 1회(초반 완화)
- IAP는 소프트런치 이후 판단(스킨팩/광고제거 등)
1. **게임 씬 구성(홈 → 인게임 → 결과 등)**:
- 홈(레벨 선택/간단 설정) → 인게임(퍼즐+상단 풍선 UI) → 결과(클리어/실패, 보상/리트라이) → 다음 레벨

### **4. 핵심 기믹**

**주요 기믹 설명**:
- 모든 화살표는 **점(dot) 그리드** 위에 배치(색/길이/방향 랜덤).
- 플레이어는 화살표를 이동시켜 **그리드 경계를 벗어나게(Extract)** 만들며,
- **그리드를 벗어난 화살표는 더 이상 화살표끼리 충돌하지 않고**,
- 즉시 **가장 가까운 동일 색 풍선**을 자동 타겟팅해 **호밍 이동 후 Pop**.
- 핵심은 “탈출 순서/색상 관리”가 곧 “상단 목표 제거 전략”이 되는 구조.
**차별점 및 재미 요소**:
- 기존 ArrowEscape의 “빼내기 퍼즐”을 **‘공격/파괴 결과’로 확장**
- 퍼즐의 정답이 **즉시 시각적 보상(풍선 팝/컨페티/콤보)**로 연결
- 색상 매칭으로 규칙이 직관적이며, “어떤 색을 먼저 빼낼지”가 전략 포인트

---

### **5. 재미 요소**

**유저 유인 요인(감성·전략·수집·성장 등)**:
- **만족감(Satisfying)**: 풍선 팝, 컨페티, 콤보 연출(리듬감)
- **직관성(3초 문법)**: “빼면(탈출) → 같은 색이 터진다”
- **가벼운 전략**: 색상/순서 최적화(막힘 해소의 ‘퍼즐 해법’ 느낌)
- **시선 안정성**: 풍선은 **상단 고정 UI(20~25%)**로 항상 보여 인과관계 유지

---

### **6. 흥행 가능성 근거**

**비교·벤치마킹 사례**:
- **ArrowEscape류 퍼즐**(방향/공간 해소/탈출) 감성
- **Balloon Pop / Pop류**의 즉시 보상(파괴/컨페티/ASMR 계열 만족감)
**시장 차별화·우위 포인트**:
- 단순 클론이 아니라 **“퍼즐 결과가 곧 타겟 파괴로 이어지는 결과 레이어”**를 추가
- UA 관점에서 한 컷 이해 가능: **퍼즐에서 화살표를 빼내면 위 목표가 터짐**
- “상단 고정 목표 UI + 하단 스크롤 퍼즐” 구조로 큰 레벨에서도 인과 유지 가능

### **7. MVP 기준 3개월 제작 가능성**

**MVP 핵심 기능**:
- Dot-grid 좌표/스냅 시스템
- 화살표 오브젝트(색/길이/방향/상태) 및 그리드 내 이동/막힘 판정
- Extract 판정(그리드 경계 이탈 시 상태 전환)
- 타겟팅: 가장 가까운 동일 색 풍선 선택(리스트 관리 + 거리 계산)
- 연출 기반 호밍(Bezier 이동) + 타겟 락온(글로우/링)
- 풍선 팝(컨페티/사운드/콤보) + 승/패 판정 + 리트라이
- 상단 Target UI 고정(권장 비율 20~25%) + 기본 UI(리셋/힌트)
**1개월 내 구현 근거**:
- 핵심 로직이 **“상태 전환(온그리드→추출) + 타겟 선택 + 연출 이동 + 팝”**으로 단순하며, 물리 시뮬레이션 의존도가 낮음
- 풍선은 월드가 아닌 **UI 레이어**로 고정하여 카메라 이슈(듀얼 카메라/PIP)를 회피
- 프로토타입은 2~4주 내 핵심 재미(인과/보상) 검증 가능, 이후 3개월 MVP에서 레벨/튜토리얼/수익화 안정화 가능
**MVP 제외 가능 기능**:
- 복잡한 메타(성장 트리, 장비, 재화 경제)
- 특수 화살표/특수 풍선(분열/폭발/이동 풍선 등)
- 정교한 스킨 컬렉션/상점(소프트런치 이후)
- 듀얼 카메라/PIP 미니뷰(인지 부하·복잡도 증가로 비권장)
