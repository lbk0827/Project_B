## 2. 테마와 화면 설명

### 2-1. 테마와 스토리

**비주얼 컨셉**
- 클린하고 미니멀한 하이퍼캐주얼 스타일
- 밝고 선명한 색상 팔레트 (파스텔 배경 + 비비드 포인트 컬러)
- 화살표: 폴리라인(꺾이는) 형태, 색상별 명확한 구분 (빨강, 파랑, 초록, 노랑, 보라)
- 풍선: 광택감 있는 3D 느낌의 2D 스프라이트, 반사광 하이라이트

**비주얼 경쟁력 요소**
- 풍선 팝 시 컨페티 폭발 이펙트 (ASMR 계열 만족감)
- 화살표 호밍 시 곡선 궤적의 시각적 쾌감
- **Snake 방식 이동**: 머리가 먼저 이동하고 몸통이 따라오는 자연스러운 움직임

**인기 유발 요소**
- ASMR 만족감: 풍선 터지는 사운드 + 컨페티 비주얼
- 색상 매칭의 직관성: "탭하면 같은 색이 터진다" (3초 문법)
- UA 크리에이티브 강점: 1컷으로 게임 이해 가능

**테마/스토리**
- MVP에서는 스토리 없이 순수 퍼즐에 집중
- 확장 시: "파티 준비", "축제 장식" 등 가벼운 테마 적용 가능

- 폴리라인(꺾이는) 화살표 지원

**메타/아웃게임 UI 구성 (MVP)**
- **홈 화면**: 레벨 선택 (간단한 진행 맵 형태)
- **설정 화면**: 사운드 On/Off, 진동 On/Off
- **결과 화면**: 클리어/실패 표시, 별점(1-3성), 다음 레벨/리트라이 버튼

**게임 플레이 기본 흐름 (업데이트됨)**
1. 레벨 진입 → 퍼즐 영역(화살표) 관찰
2. 화살표를 **탭**하면 자동으로 Head 방향으로 이동 시작
3. **Snake 방식**으로 이동 (머리가 먼저, 몸통이 따라감)
4. 다른 화살표에 막히면 정지
5. 그리드 경계를 넘으면 "탈출(Extract)" - 셀이 하나씩 빠져나감
6. 탈출한 화살표가 동일 색상 풍선으로 호밍 → Pop
7. 모든 목표 풍선 파괴 시 클리어 / 화살표 소진 시 실패

**핵심 기믹 (업데이트됨)**
- **폴리라인 화살표**: 여러 세그먼트로 구성된 꺾이는 화살표
- **탭 기반 이동**: 드래그 대신 탭으로 화살표 발사
- **Snake 방식 이동**: 머리가 먼저 이동, 몸통이 따라옴
- **충돌/막힘**: 다른 화살표에 막히면 이동 불가
- **탈출(Extract)**: 그리드 경계 이탈 시 셀이 하나씩 빠져나감
- **색상 매칭 호밍**: 탈출 화살표 → 가장 가까운 동일 색 풍선 자동 타겟팅
- **풍선 팝**: 충돌 시 파괴

**아웃게임 MVP 핵심 요소**
- 레벨 진행 시스템 (1-100+ 레벨)
- 별점 시스템 (이동 횟수 기반 1-3성)
- 로컬 저장 (PlayerPrefs)

## 3. 구현 상세

### 장르 분석: ArrowEscape + Target Pop 하이브리드

**ArrowEscape 장르 특징**
- 여러 화살표가 겹쳐있는 공간에서 순서대로 빼내는 퍼즐
- 화살표는 자신의 방향으로만 이동 가능
- 공간 추론 + 순서 최적화의 전략적 재미
- 성공 사례: Arrow Escape 시리즈, Parking Jam 류

**Target Pop 장르 특징**
- 목표물(풍선, 공 등)을 터뜨리는 즉각적 만족감
- ASMR 계열의 감각적 피드백 (사운드, 비주얼)
- 성공 사례: Balloon Pop, Pop It 류

**하이브리드 설계 의도**
- ArrowEscape의 **전략적 깊이** + Pop의 **즉각적 보상**
- 퍼즐 해결의 결과가 시각/청각 보상으로 직결
- "어떤 순서로 빼야 할까?" + "빼면 터진다!" = 복합 만족감

### 3-1. 게임의 기본 룰

#### 인게임 기본 룰 (업데이트됨)

**1. 그리드 시스템**
- Dot-grid 기반 정수 좌표계
- 화살표는 점과 점 사이에 정렬 (스냅)
- 그리드 크기: 레벨별 가변 (5x5 Easy ~ **30x30** Hard)
- **Dot 표시**: 화살표가 차지하는 위치에만 표시 (전체 그리드 X)
- **카메라 자동 조절**: 레벨 로드 시 그리드 크기에 맞춰 자동 줌

**2. 화살표(Arrow) 속성 (업데이트됨)**
| 속성 | 값 | 설명 |
|------|-----|------|
| 색상 | Red, Blue, Green, Yellow, Purple | MVP 5색 |
| **세그먼트** | SegmentData[] | 꺾이는 경로 정의 |
| **Head 방향** | Up, Down, Left, Right | 마지막 세그먼트 방향 |
| 상태 | Idle, Moving, Extracted | 3가지 상태 |

**3. 세그먼트 데이터 구조 (신규)**
```csharp
[System.Serializable]
public class SegmentData
{
    public ArrowDirection direction;  // Up(0), Down(1), Left(2), Right(3)
    public int length;                // 1~5칸
}

// 예시: ㄱ자 화살표
segments = [
    { direction: Right, length: 3 },  // 오른쪽으로 3칸
    { direction: Up, length: 2 }      // 위로 2칸 (Head 방향)
]
```

**3. 이동 규칙 (업데이트됨)**
- **탭 기반**: 화살표를 탭하면 Head 방향으로 자동 이동 시작
- **Snake 방식**: 머리가 먼저 이동, 꼬리가 따라옴
  - 꼬리 셀 제거 → 머리 앞에 새 셀 추가 → 반복
- 이동 중 다른 화살표와 충돌하면 즉시 정지
- 꼬리가 비워질 예정인 위치로는 이동 가능 (자기 꼬리 통과)
- 그리드 경계를 벗어나면 "탈출(Extract)" 트리거

**4. 탈출 및 타겟팅 규칙**
- 탈출 시작: Head가 그리드 밖으로 나가면 시작
- **셀 단위 탈출**: 셀이 하나씩 그리드 밖으로 빠져나감
- 마지막 셀까지 나가면 탈출 완료 → 호밍 화살표 생성
- 자동으로 가장 가까운 동일 색상 풍선을 타겟팅
- Bezier 곡선으로 호밍 이동 → 풍선 충돌 → Pop

**5. 풍선(Balloon) 규칙**
- 화면 상단 고정 영역에 배치 (Screen Space)
- 색상별로 구분, 동일 색상 화살표만 파괴 가능
- 파괴 시: 스케일 애니메이션 + 컨페티 + 사운드

**6. 승리/패배 조건 (업데이트됨)**
- **승리**: 모든 목표 풍선 파괴
- **패배**: 모든 화살표 소진 후에도 풍선이 남음
- **별점 기준**: 이동 횟수 (Par 이하 3성, Par+1 2성, Par+2 이상 1성)

#### 아웃게임 기본 룰 (MVP)

**1. 레벨 진행**
- 선형 진행 (Level 1, 2, 3...)
- 클리어 시 다음 레벨 해금
- 별점 기록 저장 (1-3성)

**2. 설정**
- BGM On/Off
- SFX On/Off
- 진동 On/Off

#### 3-3-2. Snake 이동 메커니즘 (신규)

**Snake 이동 로직**
```csharp
// 의사 코드
void SnakeMove(Vector2Int nextHeadPos)
{
    // 1. 꼬리 셀 점유 해제
    Vector2Int tailPos = occupiedCells[0];
    GridSystem.SetOccupied(tailPos, false);

    // 2. 새 머리 위치 점유
    GridSystem.SetOccupied(nextHeadPos, true);

    // 3. 셀 리스트 업데이트
    occupiedCells.RemoveAt(0);      // 꼬리 제거
    occupiedCells.Add(nextHeadPos); // 머리 추가
    headPosition = nextHeadPos;

    // 4. 비주얼 업데이트 (애니메이션)
    UpdateLineRenderer();
}
```

**충돌 검사 (Snake 방식)**
```csharp
bool CanMoveTo(Vector2Int nextHeadPos)
{
    // 자신의 꼬리 위치면 이동 가능 (Snake 방식)
    if (nextHeadPos == occupiedCells[0])
        return true;

    // 자신의 다른 몸통과 충돌하면 불가
    for (int i = 1; i < occupiedCells.Count; i++)
    {
        if (nextHeadPos == occupiedCells[i])
            return false;
    }

    // 다른 화살표와 충돌 체크
    return !GridSystem.IsOccupied(nextHeadPos);
}
```

void ExtractOneCell()

BalloonController FindTargetBalloon(Vector2 position, GameColor color)
{
    BalloonController closest = null;
    float closestDistance = float.MaxValue;

    foreach (var balloon in balloons)
    {
        if (balloon.Color != color) continue;
        if (balloon.State != BalloonState.Active) continue;

        float dist = Vector2.Distance(position, balloon.transform.position);
        if (dist < closestDistance)
        {
            closestDistance = dist;
            closest = balloon;
        }
    }

    return closest;
}

#### 3-3-5. 호밍 이동 메커니즘

**Bezier 곡선 기반 이동**
```csharp
// 2차 Bezier 곡선
Vector2 QuadraticBezier(float t, Vector2 p0, Vector2 p1, Vector2 p2)
{
    float u = 1 - t;
    return u * u * p0 + 2 * u * t * p1 + t * t * p2;
}
```

#### 역방향 채우기 알고리즘 (Reverse Filling)

**문제 해결**
- 무작위 배치 시 풀 수 없는 Deadlock 발생 방지
- 그리드 밖에서 안쪽으로 화살표를 "거꾸로" 쏜다고 가정

**알고리즘 절차**
```
1. 빈 그리드 생성 (PNG 실루엣 또는 수동 설정)
2. 경계 셀에서 시작 (역방향 채우기)
   - 그리드 가장자리 또는 무효 셀과 인접한 셀 선택
   - 해당 셀의 탈출 방향 결정
3. 화살표 성장
   - 탈출 방향의 반대로 셀 추가
   - 확률적으로 꺾임 발생 (폴리라인)
   - 최소/최대 길이 제한 적용
4. 세그먼트 계산 및 저장
5. 반복: 모든 유효 셀이 채워질 때까지
6. 검증: BFS Solver로 풀이 가능 여부 확인
```

**파라미터**
| 파라미터 | 기본값 | 설명 |
|----------|--------|------|
| 최소 화살표 길이 | 2 | 너무 짧은 화살표 방지 |
| 최대 화살표 길이 | 8 | 너무 긴 화살표 방지 |
| 색상 수 | 5 | Red, Blue, Green, Yellow, Purple |
| 꺾임 확률 | 30% | 폴리라인 생성 확률 |
| 시드 | 랜덤 | 재현 가능한 생성을 위한 시드 값 |

#### 이미지(.PNG) → 레벨 변환

**변환 프로세스**
```
1. PNG 이미지 로드
2. 픽셀 해상도 → 그리드 크기 매핑
   - 이미지 크기를 그리드 크기로 리샘플링
3. 알파값 분석
   - 알파 > 0.5 → 유효 셀 (화살표 배치 가능)
   - 알파 <= 0.5 → 무효 셀 (빈 공간)
4. 유효 셀 그리드 생성
5. 역방향 채우기로 화살표 자동 생성
6. 검증 및 조정
7. LevelData JSON 생성
```

**이미지 규격**
- 권장 해상도: 그리드 크기와 동일 (예: 15x15 그리드 → 15x15 픽셀)
- 지원 포맷: PNG (투명도 지원)

#### 3-8-2. Hint

**기능**: 현재 탈출 가능한 화살표 하나를 하이라이트하여 힌트 제공

**동작 플로우**
```
[Hint 버튼 탭]
    ↓
[탈출 가능 화살표 탐색]
    ├── 탈출 가능한 화살표 없음 → [안내 메시지] → 아이템 미차감
    │
    └── 탈출 가능한 화살표 발견
            ↓
        [아이템 차감] - BoosterData.UseItem("Hint")
            ↓
        [해당 화살표 하이라이트 연출]
        (반짝임/글로우 이펙트, DOTween 루프)
            ↓
        [하이라이트 해제 대기]
            ├── 사용자가 아무 화살표를 탭 → 해제
            ├── 화살표 이동 시작 → 해제
            └── 일정 시간(5초) 경과 → 자동 해제
```

**탈출 가능 판정 로직**
```csharp
// 의사 코드
bool CanArrowEscape(ArrowController arrow)
{
    Vector2Int headPos = arrow.HeadPosition;
    Vector2Int direction = arrow.HeadDirection;

    // Head 방향으로 이동 시뮬레이션
    Vector2Int checkPos = headPos + direction;

    // 이동 경로에 다른 화살표가 없는지 확인
    while (GridSystem.IsInBounds(checkPos))
    {
        if (GridSystem.IsOccupiedByOther(checkPos, arrow))
            return false;
        checkPos += direction;
    }

    return true; // 경로가 비어있으면 탈출 가능
}
```

**하이라이트 연출**
- DOTween 기반 반복 스케일 펄스 (1.0 → 1.1 → 1.0, 0.5초 루프)
- 또는 알파 깜빡임 (1.0 → 0.5 → 1.0)
- 카메라가 해당 화살표를 포커싱 (자동 패닝)

**예외 처리**
- 탈출 가능 화살표가 여러 개인 경우: 가장 짧은 경로의 화살표 우선
- Deadlock 상태 (아무 화살표도 탈출 불가): 안내 팝업 표시

#### 3-8-3. Eraser

**기능**: 선택한 화살표 하나를 강제로 탈출(Extract)시킴

**동작 플로우**
```
[Eraser 버튼 탭]
    ↓
[선택 모드 진입]
    - UI 오버레이 표시
    - 안내 텍스트: "탈출시킬 화살표를 선택하세요"
    ↓
[사용자 입력 대기]
    ├── 화살표 탭
    │       ↓
    │   [아이템 차감] - BoosterData.UseItem("Eraser")
    │       ↓
    │   [강제 탈출 실행]
    │       ↓
    │   [그리드에서 화살표 셀 즉시 제거]
    │       ↓
    │   [호밍 화살표 생성]
    │   (탈출 방향 = 화살표의 Head 방향)
    │       ↓
    │   [풍선 팝] - 기존 탈출 프로세스와 동일
    │       ↓
    │   [선택 모드 종료]
    │
    ├── 빈 영역 탭 → [선택 모드 종료] (아이템 미차감)
    │
    └── 취소 버튼 탭 → [선택 모드 종료] (아이템 미차감)
```

**강제 탈출 메커니즘**
```csharp
// 의사 코드
void ForceExtract(ArrowController arrow)
{
    // 1. 화살표가 점유한 모든 셀 해제
    foreach (var cell in arrow.OccupiedCells)
    {
        GridSystem.SetOccupied(cell, false);
    }

    // 2. 호밍 화살표 생성 (기존 탈출 프로세스 활용)
    GameManager.SpawnHomingArrow(arrow);

    // 3. 원본 화살표 제거
    arrow.ForceDestroy();

    // 4. 이동 횟수 증가하지 않음
}
```

**상세 규칙**
- 이동 중(Moving 상태)인 화살표는 선택 불가
- 이미 탈출 중(Extracting 상태)인 화살표는 선택 불가
- Idle 상태의 화살표만 선택 가능
- Eraser 사용은 이동 횟수에 포함되지 않음 (별점에 영향 없음)
- 강제 탈출 시 다른 화살표와의 충돌 무시 (경로 상의 화살표를 관통)

**선택 모드 UI**
- 반투명 오버레이 (화살표 영역만 밝게)
- 상단에 안내 텍스트
- 우측 상단에 취소(X) 버튼

## 4. 경쟁력 제안

### 4-1. 핵심 경쟁력

**핵심 차별화 요소**
1. **퍼즐 + 즉각 보상의 결합**: ArrowEscape의 전략적 재미 + Pop의 감각적 만족감
2. **Snake 방식 이동**: 자연스럽고 직관적인 화살표 움직임
3. **폴리라인 화살표**: 단순 직선이 아닌 꺾이는 화살표로 전략 깊이 증가
4. **명확한 인과관계**: "탭하면 터진다"는 3초 내 이해 가능
5. **UA 크리에이티브 강점**: 한 컷으로 게임 이해 가능

**기존 경쟁 게임과의 차이점**
| 구분 | Arrow Escape 류 | Pop 류 | Arrow Pop Escape |
|------|----------------|--------|------------------|
| 목표 | 화살표 빼내기 | 타겟 터뜨리기 | 빼내서 터뜨리기 |
| 조작 | 드래그 | 탭 | **탭 (자동 이동)** |
| 화살표 | 직선 | 없음 | **폴리라인 (꺾임)** |
| 이동 | 즉시 | 없음 | **Snake 방식** |
| 만족감 | 퍼즐 해결 | ASMR | 퍼즐 + ASMR |

**타겟 유저와 시장 적합성**
- **주요 타겟**: 15-35세 캐주얼 퍼즐 유저
- **성향**: 짧은 세션 선호, 빠른 보상 추구, 스트레스 해소 목적
- **지역**: 글로벌 (특히 북미, 유럽, 동남아)
- **시장**: 하이퍼캐주얼 퍼즐 시장 (검증된 ArrowEscape + Pop 조합)

### 4-2. 경쟁 서비스 분석

**주요 경쟁 게임 리스트**

| 게임 | 장르 | 퍼블리셔 | 특징 |
|------|------|----------|------|
| Arrow Escape | 빼내기 퍼즐 | Voodoo | 화살표 빼내기 원조 |
| Parking Jam 3D | 빼내기 퍼즐 | Popcore | 차량 빼내기, 3D |
| Water Sort Puzzle | 정렬 퍼즐 | IEC Global | 색상 정렬 |
| Balloon Pop | 타겟 Pop | Various | 풍선 터뜨리기 |

**Arrow Pop Escape 포지셔닝**
- Arrow Escape의 전략성 + Balloon Pop의 만족감 + **Snake 이동의 직관성**
- 예상 D1 Retention: 35-40%
- 예상 CPI: $0.25-0.35
